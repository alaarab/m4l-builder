# UI Validation Notes

This directory stores structured validation notes for major Ableton UI sprints.

Use `docs/ableton_ui_validation_template.md` for every new note.

Related overview docs:

- `docs/ableton_ui_tracker.md`
- `docs/ableton_ui_scoreboard.md`

## Naming

Use week-based or sprint-based filenames:

- `week_03_parametric_eq.md`
- `week_05_linear_phase_eq.md`
- `week_07_spectrum_analyzer.md`
- `week_09_ladder_filter.md`
- `week_11_framework_reaudit.md`

## Required Content

Each note must cover:

- chain view readability
- expanded view readability
- one fast change
- one precise change
- one exploratory change
- graph or control drift checks
- analyzer or meter behavior when relevant
- final score using `docs/ableton_ui_review_checklist.md`

## Status Rules

- `planned`: scaffold exists, validation not yet run
- `in-progress`: partial observations exist
- `complete`: real in-Live session finished

Do not claim `complete` unless the device was tested during real playback in
Ableton Live.
