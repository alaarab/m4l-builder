## Product Role

- What job does this device solve?
- Is it a fast mix tool, precision tool, measurement tool, performance effect,
  or dual-surface instrument?
- What part of that role is obvious from the UI alone?

## Hero Surface

- What is the primary editing surface?
- Why does it earn that role?
- What is editable directly from it?

## Always Visible Vs Contextual

- What stays visible at all times?
- What becomes contextual?
- Which controls are utility controls rather than main editing controls?

## Selection Model

- How is selection shown?
- If multiple objects are editable, where is the canonical contextual editor?
- What would drift or confuse the user if copied into another surface?

## Compact Legibility

- What survives in compact or chain view?
- What is the minimum state the user still understands?
- What gets deferred to expanded view?

## State Encoding

- How are selected, enabled, bypassed, listened, warned, or monitored states
  distinguished?
- Which states affect audio vs display only?
- Which state is easiest to misunderstand?

## Motion, Analyzer, And Meter Behavior

- How does motion or animation avoid noise?
- How is analyzer or meter data made useful without becoming the main event?
- What is truthful versus smoothed or held?
